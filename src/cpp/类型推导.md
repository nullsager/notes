在大多数情况下，类型推导将从推导出的类型中删除 const

```cpp
const int a { 5 };
auto b { a };  // b的类型是int
```

如果您希望推导出的类型为 const，则必须在定义中自己提供 const：

```cpp
const int a { 5 };
const auto b { a };
```

同样的，constexpr由于含有隐式const属性，进行类型推导时也会丢弃掉const属性

---

字符串字面量会推导为`const char*`，而不是`std::string`

```cpp
auto s { "Hello, world" }; // s的类型是const char*
```

如果希望从字符串字面量推导出的类型为 `std::string` 或 `std::string_view` ，需要使用 s 或 sv 字面量后缀

```cpp
int main() {
    using namespace std::literals;
    auto s1 { "goo"s };  // "goo"s is a std::string literal, so s1 will be deduced as a std::string
    auto s2 { "moo"sv }; // "moo"sv is a std::string_view literal, so s2 will be deduced as a std::string_view
}
```
